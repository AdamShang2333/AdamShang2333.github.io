# swagger 使用文档

# 1 什么是 swagger

简而言之 : swagger 是一个规范和完整的框架，用于实时生成、描述、调用和可视化 RESTful 风格的 Web 服务。
　　Swagger== 采用 OpenAPI 规范 ==，OpenAPI 规范这类 API 定义语言能够帮助你更简单、快速的表述 API，尤其是在 API 的设计阶段作用特别突出。
　一旦编写完成，API 文档可以作为：

・需求和系统特性描述的根据

・前后台查询、讨论、自测的基础

・部分或者全部代码自动生成的根据

・其他重要的作用，比如开放平台开发者的手册

# 2 使用 swagger 编写 API 文档

swagger 编写 API 文档有以下几种方式:

  1. 定义 Yml 格式文件 编写文档
  2. 定义 Json 格式文件 编写文档
  3. 使用配置及少量代码 实现文档编写
    • 在实际开发中 推荐以整合进 SpringBoot 的注解形式完成文档开发
  • 关于 Yml 和 Json 格式开发静态文档 以及 OpenAPI
    可以参考: 如何编写基于 OpenAPI 规范的 API 文档

####2.4 项目中使用swagger注解
#####使用示例

Controller接口信息
```java
/**
 * 队伍经营数据
 * @param inDto
 * @return
 */
@ApiOperation("获取队伍经营数据")
// !!! swagger不会生成特定httpMethod的api,而是多组Method的api
// 不在@ApiOperation的httpMethod属性中赋值 使用@PostMapping等更精准的注解即可
@RequestMapping("/getTeamOperation")
public ExecuteResult<TeamOutDto> getTeamOperation(@RequestBody @Validated TeamInDto inDto ){
    return cockpitService.getTeamOperation(inDto);
}
/**
 * 获取机构业绩排行榜
 * @param inDto
 * @return
 */
@ApiOperation(value = "获取指定等级的业绩排名情况",httpMethod = "POST",response = ComApiDto.class)
@RequestMapping("/getTeamRank")
// !!! ExecuteResult未加泛型 对于swagger 不能真正的推断到返回的result中的dto 
//      ==> api中的result信息是空的
//  解决: 添加返回值的泛型  或者在注解中对response属性赋值即可
public ExecuteResult getTeamRankByGrade(@RequestBody @Valid ComRankInDto inDto){
    return cockpitService.getTeamRankByGrade(inDto);
}

注意点:
1 @ApiOperation 注解，可以只填写一个值，默认为 value 值
===> 此时 所有的其他信息根据方法的返回值和使用的注解推断
2 未在 @ApiOperation 中指明 HttpMethod 和 respones 的时候 注意:

  1. 不要使用 @RequestMapping 注解 会生成重复的 api 信息 (get post put delete ....)
  2. 返回值的 ExecuteResult 一定要指明泛型类型 不然无法将真正的返回 dto 填入 api 信息

# 2.5 swagger 中常用的注解

@Api 　标记在 controller 上面，描述整个 controller

属 性 名 称 作 用
**String value ** url 的路径值

String protocols 接口支持的协议 Possible values: http, https, ws, wss
Authorization[ ] authorizations 高级特性 认证时配置
boolean hidden 是否在文档中隐藏 默认为 false

属 性 名 称 作 用

String[] tags 操作标签，非空时将覆盖 value 的值

String httpMethod 指定 HTTP 方法 <br> 只能键入 "GET", "HEAD", "POST", "PUT", "DELETE", "OPTIONS" and "PATCH" ; <br > 注：如果不输入 则会根据 @PostMapping 等来填入合适的值  <br> 不推荐使用 @RequestMapping 会导致创建所有类型的重复 api 文档

String protocols 接口支持的协议；Possible values: http, https, ws, wss
Authorization[ ] authorizations 获取授权列表（安全声明），如果未设置，则返回一个空的授权值。

# 3 swagger 的文档导出

swagger 本身不支持文档的导出功能，
但默认情况下 请求 http://ip:post/v2/api-docs 可以获取到所有 API 的 Json 数据，
swagger 也是渲染该接口的数据形成的可视化 api 界面
为 api 的导出提供了可能

注：获取 api 的 json 数据地址可以在配置文件中更改:
springfox.documentation.swagger.v2.path 的值来指定接口

使用中需要导出 swagger 文档的时候 一般有以下几种方案 :
1 使用 swagger2markup 导出文档 (markdown , asciidoc , wiki)
2 使用 swagger2word 导出文档 (word)
3 使用其他前端带有导出功能的前端 ui 界面
　　本文推荐使用 knife4j (导出 markdown , html , word , pdf)
　　注: knife4j 的 word 和 pdf 导出功能暂未开发完成
　　==> 可通过文档转换神器 pandoc 获取任意格式文档

# 3.1 使用 swagger2markup 导出文档

# 相关依赖

swagger2markup 插件参数说明：
configuration.swaggerInput ：
　　该标签内容需修改为需要导出接口项目的 /v2/api-docs 路径
configuration.outputFile ：
　　该标签为生成单个文档指定文档生成路径，如生成 txt、md 等文件，可随意修改；
　　但注意，该标签与 outputDir 标签二选一；
configuration.outputDir ：
　　该标签为生成多个文档指定文档目录，如生成 ASCIIDOC 文件，该类文件可用于结合 asciidoctor 插件生成 html 文件；
configuration.config ：
　　该标签内定义的 swagger2markup.markupLanguage 子标签，只能同时存在一个，
　　如指定生成 markdown 即 md 文件时，就不能指定生成其他类型；

注意：指定生成 ASCIIDOC 文件类型时，需与 configuration.outputDir 标签配合使用；

# 使用 mvn 命令导出

生成markdown或wiki格式

1 运行当前项目
2 确认系统的 /v2/api-docs 接口可用
3 进入项目根目录，进入 cmd ，执行 mvn 命令：
mvn swagger2markup:convertSwagger2markup
4 结果确认：将在 configuration.outputDir 或 configuration.outputFile 标签指定的目录和文件，生成对应的接口文档；

生成html格式(需要使用ascii格式文件 配合asciidoctor)

在利用 swagger2markup 生成了 ASSIIDOC 文件之后，
将 configuration.sourceDirectory 指定到 ASSIIDOC 文件的目录
进入到项目根路径，使用 maven 命令： mvn asciidoctor:process-asciidoc

knife4j 的前身是国人开发的 swagger-bootstrap-ui
是 springfox-swagger 的增强 UI 实现，
为 Java 开发者在使用 Swagger 的时候，能拥有一份简洁、强大的接口文档体验
