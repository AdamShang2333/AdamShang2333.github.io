# SpringMvc 源码浅析

# Spring MVC 的运行顺序

springMvc的关键入口 是DispatcherServlet

Model View Controller , web 层的框架 与 controller , service,dao 的三层架构不是一个概念

  • 1 初始化

  1. web.xml 配置 DispatcherServlet

  2. 加载 IOC 容器 (ApplicationContect)

  3. 开始扫描 SpringMVC 配置，一般来说 扫描 :

​ 注解，view 配置，拦截器 转换器 视图解析器

  1. 解析成一个 HandlerMapping 的 List (主要是保存了 url 和具体执行方法的对应关系)
  • 2 等待用户请求
  1. 浏览器输入 URL
  2. 统一拦截 (扫描 HandlerMapping)
  3. DispatcherServlet 接收请求，使用 HandlerMapping 找到对应的响应方法执行
  4. 响应结果返回
