# SpringBoot 注解总结

# 1 组件注册注解

# @Configuration

标注在类上，标识为配置类

# @Bean

标注在方法上，向容器注入 Bean ;

可以传入 value 属性的值来指定 Bean 的名字

@Bean 标注的方法创建对象时，方法参数的值默认从 IOC 容器中获取 (可以不加 @Autowired 指定)

# @ComponentScan

标注在配置类上，配置组件的扫描规则

可以在一个类上标注多个，指定复合的扫描规则

FilterType 有多种:

ANNOTATION : 根据注解

ASSIGNABLE_TYPE : 根据类的类型

REG : 根据正则表达式

ASPCJ : 根据 aspcJ 表达式

CUSTOM : 根据用户自定义规则

==> 实现 TypeFolter 接口的 match 方法来灵活自定义过滤

# @Scope

标注在类或方法，指定 bean 的作用范围

可选值:

SCOPE_PROTOTYPE // 多实例

​ ==> 创建 IOC 容器是不会创建对象，每次获取对象时才会调用方法创建对象

SCOPE_SINGLETON // 单实例 (默认值)

​ ==> 容器启动时就会调用方法创建对象放入 IOC 容器中

SCOPE_REQUEST //web 同一次请求创建一个实例
SCOPE_SESSION //web , 同一个 session 创建一个对象

# @Lazy

标注在类，方法，构造方法，属性，指定对象的创建使用懒加载

# @Conditional

标注在 类，方法

按照一定条件，满足条件的 bean 注册到 IOC 容器中

在 Spring 的底层中大量使用

传入实现 Condition 接口，match 方法的子类来完成自定义的匹配条件

# @Import

标注在类上，导入外部第三方包中的组件到 IOC 容器中

bean 的 id 默认是类的全类名

可以实现 ImportSelector 接口，来重写 selectImports

实现自定义的组件导入逻辑

ImportSelector 接口:

ImportBeanDefinitionRegistrar 接口 :

# FactoryBean 接口

factoryBean 接口也能实现向 IOC 容器中注册 bean ,

通过实现接口，重写

T getObject ( )

Class<?> getObjectType( )

boolean isSingleton()

重写 factoryBean 中的方法，返回 getObject () 中的工厂创建对象，

使用 applicontex.getBean (工厂类名) 获取的是工厂 bean 生产的 bean;

要获取工厂 bean 本身，则在类名前加 "&" 前缀来指明获取工厂 bean

# Bean 生命周期

bean 的生命周期，指的是 bean 创建 --> 初始化 --> 销毁的过程

一般 由 IOC 容器来管理整个 bean 的生命周期

可以自定义初始化和销毁的方法，容器在 bean 进行到当前生命周期时，调用我们自定义的方法来执行

# 自定义 @bean 的 initMethod/destroyMethod

# @Bean(initMethod="" )

指定 bean 的初始化方法，在对象创建完成 并赋值好属性之后调用

# @Bean(destroyMethod="" )

指定 bean 的销毁方法，在 IOC 容器关闭后调用

多实例 Bean 的情况下，

在对象被获取后创建完成时 调用 init 方法；

在 IOC 容器销毁时并不会销毁多例 Bean (交由用户自行销毁 IOC 容器不会管理 bean 的完整生命周期)

# InitializingBean 接口

###DisposableBean 接口

# @PostConstruct

JSR250 规定，标注在方法上，bean 创建完成并赋值后调用

# @PreDestroy

JSR250 规定，标注在方法上，在容器销毁之前调用

# BeanPostProcessor 接口

bean 的后置处理器，在 bean 初始化前后进行一些处理工作

常用的 ApplicationContextAwaewProcessor 接口，也是实现了 BeanPostProcessor 接口，才能在实现类中传入当前的 applicatonContext 对象

# 属性赋值

# @Value

对属性赋值

可以写:

​ 1. 基本类型的数值

2.	可以写SpEL , #{}
3.	可以写	${ } , 去除配置文件(properties/yml)中的值

# @PropertySource

指定配置文件路径，可重复标注多个 source

加载外部配置文件中的 k/v 值 保存在运行时的环境变量中；

加载完外部的配置文件以后，使用 ${ } 取出配置文件的值

也可以使用 applicationContext 对象的 getEnvironment 方法

获取到 environment 对象后，使用 environment.getProperty 来获取配置文件 k/v

# 自动装配

# @Autowired

标注在属性，参数，方法，构造方法上；实现依赖注入 (DI)

默认按照类型在 IOC 容器找到组件进行装配，

如果找到多个相同类型的组件，再将属性名称作为组件 id 去完成装配.(可以使用 @Qualifier 指定)

注解属性 :

required : 默认 true ,IOC 容器中必须有相应 bean 来装配

# @Primary

标注在 类型，方法上；指定默认的首选装载 bean

@Primary 和 @Qualifier 同时使用时

会优先按照 @Qualifier 执行装配

# @Resource

JSR250 规范注解，不支持 @Primary @Qualifire

默认按照属性名称来装配，可以通过 name 属性来指定装配 bean

# @inject

JSR330 规范注解，需要额外导入 javax.inject 包；

和 Autowired 功能一样，能支持 @Primary ;

但是没有 required =false 属性

# Aware 接口

Bean 在装配时，如果想使用 Spring 底层的组件 则相应的 Bean 实现 xxxAware 接口 即可传入相应 Spring 组件

# 常用 Aware 子接口:

  • ApplicationContextAware
    • 获取到运行时的 IOC 容器
  • BeanNameAware
    • 获取当前 bean 在 IOC 中的名字
  • EmbeddedValueResolverAware
    • 获取 StringValueResolver (占位符解析器)
  • ApplicationEventPublishAware
  • ResourceLoaderAware
  • MessageSourceAware

# @Profile

指定组件在哪个环境的情况下，动态的激活和切换一系列组件 (开发 生产环境的数据库等配置)

可以标注在 类，方法上 (写在类上，只有是指定环境的时候，整个配置类下的所有组件才能生效)

注：没有标注环境标识的 bean, 在任何环境下都是加载的

配置 @profile 注解环境的方式

  1. 使用命令行参数：在虚拟机参数位置加载 - Dspring.profiles.active=test

  2. 使用代码 API 设置

    // 1.创建application对象
new ApplicationContext()
// 2.environment对象设置activeProfile属性
context.getEnvironment().setActiveProfiles("test","dev");
// 3.注册主配置类 (MyConfigOfProfile 为自己配置的环境配置类)
context.redister(MyConfigOfProfile.class);
// 4.启动刷新容器
context.refresh();
