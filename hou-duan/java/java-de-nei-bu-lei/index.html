# 前言 -- 类的成员

一个类的成员，包含如下内容:
属性
方法
类
静态代码块

内部类可以看做一个外部类的属性，
在 Java 中，可以将一个类定义在另一个类里面或者一个方法里面，这样的类称为内部类。与之对应，包含内部类的类被称为外部类。
优势：内部类提供了更好的封装，可以把内部类隐藏在外部类之内，通过访问修饰符控制其他类对内部类的访问，更好的实现了信息隐藏
分类：成员内部类、静态内部类、方法内部类、匿名内部类

# 内部类的分类

内部类可以分为:
普通内部类
静态内部类
匿名内部类
局部内部类

# 普通内部类

特点 :

  1. 内部类可以直接访问外部类的成员；
    　　如果出现同名属性或者方法，优先访问内部类中定义的 (默认存在 this)
  2. 可以使用 外部类.this. 成员 的方式，访问外部类中同名的信息 (成员属性和成员方法)
  3. 外部类访问内部类信息，需要通过内部类实例，无法直接访问
  4. 一个外部类可以有多个内部类对象 (引用的内存地址不同)
    　　注：当一个类还有内部类引用的时候，这个类不会被 JVM 回收
    　　　　　　可以类比为数据库的外键约束，有引用的情况下不能被删除

# 静态内部类

我们知道， 一个类的静态成员独立于这个类的任何一个对象存在.
只要在具有访问权限的地方，
我们就可以通过 类名。静态成员名 的形式来访问这个静态成员，

同样的，静态内部类也是作为一个外部类的静态成员而存在，
创建一个类的静态内部类对象不需要依赖其外部类对象。

特点 :

  1. 创建无需依赖外部类对象，通过类来创建即可
  2. 只能访问外部类的静态成员变量
  3. 外部类依然可以访问静态内部类对象的所有访问权限的成员
