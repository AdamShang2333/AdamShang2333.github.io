# Collection 容器

容器的特点:
1. 只用于存储对象 --> 但可以使用泛型规定具体对象
--> 如果是基本类型，在 Java 内部也会转为对应类的对象
2. 长度可变
3. 存储对象的类型可以不同 --> 无泛型时

# 容器方法

# .add 添加一个对象

# .addAll 把参数容器的所有元素添加到容器中

// 把参数容器(a2)整体作为一个对象add进数组arr中
 arr.addAll(a2);

# .size () 获取容器的大小 (无参)

# .remove (object) 移除 (只移除遇到的第一个)

# .contains () 判断是否包含某一对象 (遇到的第一个)

.contains 返回布尔值

# 迭代器 iterator

    // 迭代器使用完之后 指针指向迭代器之前停止的位置(遍历后 指针停在最后) 无法重复利用
    // 迭代器想用时要再次赋值 !!!

 Iterator<String> it = arr.iterator();
        while (it.hasNext()) {
        // 每次.next 调用一次指针 进行一次迭代器的移位  
        // 实质上在循环内执行了两次指针的移动 对奇数次来说 最后一次会找不到可用元素
              System.out.println(it.next());           
             if ("王五".equals(it.next())) {
                it.remove();

            }
        }
        // 迭代器使用完之后 指针指向迭代器之前停止的位置(遍历后 指针停在最后) 无法重复利用
        // 迭代器想用时要再次赋值 !!!
        it = arr.iterator();

# 泛型

限制容器中保存的对象类型
每次取出的对象，想让它是规定的类型 (不是 object)
< > 内可以放入类 基本数据类型等

	List<String> l = new ArrayList<>();

# List

威力增强版的数组，特点 :
1. 元素有序 (按放入顺序)
2. 按下标取值 (从 0 开始)
3. 放入对象可重复
4. 继承自 collection

# List 特有方法

# 按下标 添加 元素

arr.add(0,"小金刚");

# 按下标 移除 元素 (注意下标越界)

 arr.remove(2);

# 按下标 修改 元素 (注意下标越界)

 arr.set(0,"太二真人");

# 按下标 获取 元素 (注意下标越界)

 arr.get(5);

# List 遍历 (三种方法)

# fori --> 循环代码需要顺序或者下标参与

 // 第一种 for循环 arr.fori
      
        for (int i = 0; i < arr.size(); i++) {
            System.out.println(arr.get(i));
        }

# for --> 循环代码只需要元素 不需要下标

 // 第二种 foreach 循环  arr.for

        // 内部的 o1 就相当于 arr.get(i)  方法替我们实现 直接使用o1就行
        for (Object o1 : arr) {

        }

# 迭代器 --> 需要在遍历期间更改 List 的长度

// 第三种 使用迭代器遍历List
        // 前两种在遍历过程中不能对数组元素做增删操作!!!

        // step1 获取容器的迭代器对象
        Iterator it = arr.iterator();

        // 返回布尔值 标示"之后还有没有值"
        // boolean b = it.hasNext();
        // step2 判断迭代器中还有没有元素
        while (it.hasNext()){
            // step3 获取下一个元素
            Object obj = it.next();
            if ("李四".equals(obj)){
                it.remove();
            }
       }

# Set

特点 :

  1. 元素无序
  2. 元素不能重复 (可以去重)
    继承自 collection

# List 遍历 (两种方法)

# for --> 循环代码只需要元素 不需要下标

  for (Object o1 : set) {

        }

# 迭代器 --> 需要在遍历期间更改 List 的长度

Iterator itset = set.iterator();
        while (itset.hasNext()){
            System.out.println(itset.next());
        }

# Map

Map -- 不是继承自 collection
1. 只能存储对象
2. 长度可变
3. 可以存储不同类型的对象

		// List 更倾向存储相同 类型&&作用的数据 -- 直接遍历做集中处理即可
        // Map  更倾向存储不同 类型&&作用的数据
        // 当数据类型复杂多变,但是又要集中处理的情况, 就可以使用MAp
        // 如果数据类型是固定的, 那么可以使用类来替代Map

# Map 独有特点:

  1. 无序
  2. 以键值对 (key-value) 的方式存储数据
  3. key 不能重复，一个 key 只对应一个 value (kay 又被称为映射) y=f (x)

# Map 方法

# .put (key,value) 存储键值对

 Map map = new HashMap();
        map.put("name","张三");
        map.put("age",14);
        
        // !!!key 值相同(name) 
        //    --> 张三会被覆盖为李四
        map.put("name","李四");

# .get (key) 获取值

 map.get("name");

# .remove (key) 移除值 (删除键值对)

 map.remove("name");

# .containsKey (key) 判断是否包含 key

 map.containsKey("name");

# .containsValue (value) 判断是否包含值

 map.containsValue("李四");

# .size () map 的大小 (一个键值对算一条数据)

 map.size();

# .isEmpty ( ) 判断是否大小为 0 -- 返回值为 Boolean

 map.isEmpty();

# .clear () 清空 Map (删除所有值)

 map.clear();

# 遍历

map 的遍历很少用到 (遍历目的是对数据做集中处理)
// 1. 只要有 key, 一定会获得 value
// --> 无 key 输出 null
// 2. map 是无序的 (无法使用 fori)
// 3. map 和迭代器没有关系 (迭代器只针对 collection 接口)

# 遍历方法 1:

        // 获取所有的key 遍历所有的key获取value 放入set

        // 拿到所有的key -- 放入set中(?: set 和 key都是不能重复的)
        Set set = map.keySet();
        for (Object key : set) {
            Object v = map.get(key);
            System.out.println(key + "的值是" + v);
        }

# 方法 2:

        // 在Map里面 每一个键值对都是一个entry对象
        // 容量很大,不确定 的时候适合用
        // 不许记忆... 很少用
        Set entrySet = map.entrySet();
        for (Object entry : entrySet) {
            Map.Entry e = (Map.Entry) entry;
            Object key = e.getKey();
            Object value = e.getValue();
            System.out.println(key+"的值是:"+value);
        }
