# 应用组成结构

注: !!! 在 JavaWeb 中，没有明显的主函数

WEB-INF 　－－＞"安全区" 后端判断有无权限获取该文件夹下的内容

WEB-INF 文件夹 是一个对前端隐藏的文件夹
即，前端没法通过访问 WEB-INF 路径访问里面的内容 -->
如果必须要访问，只能通过后台代码转发.

web.xml　－－＞配置文件设置

在 javaWeb 程序启动时，最早加载的配置文件

# web.xml

.xml 代码示例 :

<?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee http://xmlns.jcp.org/xml/ns/javaee/web-app_4_0.xsd"
         version="4.0">
      
     // 配置servlet    
     // Step 1 : servlet的本地设置
    <servlet>
        // 给每个servlet起别名(外号)
        <servlet-name>mservlet</servlet-name>  //*关联,须一致*
        // 指定servlet的类(Java类 关联)
        <servlet-class>com.oldFun.controller.MainServlet</servlet-class>
    </servlet>
    //  Step 2 : 创建和服务器的映射
    <servlet-mapping>
        // 和本地的"外号"关联
        <servlet-name>mservlet</servlet-name>  //*关联,须一致*
        // 指定访问的地址(起一个名字,别忘了"/"!!!)
        <url-pattern>/main</url-pattern>
    </servlet-mapping>
    
    <servlet>
        // 起名字(外号)
        <servlet-name>helloServlet</servlet-name>
        // 指定servlet的类
        <servlet-class>com.oldFun.controller.HelloServlet</servlet-class>
    </servlet>
    // 映射
    <servlet-mapping>
        <servlet-name>helloServlet</servlet-name>
        // 指定地址
        <url-pattern>/suibian</url-pattern>
    </servlet-mapping>

</web-app>

# Servlet

背景

sun 公司提供的一门用于开发动态 web 资源的技术
Sun 公司在其 API 中提供了一个 servlet 接口，

实现

用户若想用发一个动态 web 资源 (即开发一个 Java 程序向浏览器输出数据)
需要完成以下 2 个步骤：
　　1、编写一个 Java 类，实现 servlet 接口。
　　2、把开发好的 Java 类部署到 web 服务器中。
　　　　按照一种约定俗成的称呼习惯，通常也把实现了 servlet 接口的 java 程序，称之为 Servlet

作用

Servlet 接口就是处理网络请求的一套规范，
负责连接 web 服务器（如 tomcat 服务器)，存取数据
交互式的生成动态的 web 内容

小结

Servlet : JavaWeb 负责处理所有请求的类
每一个请求，对应一个 servlet 对象；每一个 servlet 类 只有一个对象 (单例设计模式)
－－＞ 每一个请求，对应一个 Servlet 类

# Servlet 实现方式

# 1. 通过接口实现 －－＞ 需要实现所有的方法！！！

​ 常用重要方法
​

>init 方法 : 初始化方法,只执行一次(创建servlet对象的时候执行.)  
　     作用：做一些设置和初始化工作    
>
>service 方法 : 会被执行多次  
　     作用：负责每次请求要显示的内容    
>
>destroy 方法 : 当servlet对象被销毁时执行 (服务器停止运行时, 对象被销毁)  
　     作用：做一些资源释放的操作

实例:

//** implements 来实现:
public class HelloServlet implements Servlet {

    /**
     * 该方法是servlet对象的初始化方法, 只会执行一次
     * 在创建servlet对象的时候执行.
     * 作用: 做一些设置和初始化工作
     * @param servletConfig
     * @throws ServletException
     */
    @Override
    public void init(ServletConfig servletConfig) throws ServletException {
        System.out.println("HelloServlet初始化");
    }

    @Override
    public ServletConfig getServletConfig() {
        return null;
    }

    /**
     * 这个方法会被执行多次,负责每次请求要显示的内容
     * @param servletRequest  请求
     * @param servletResponse 响应
     * @throws ServletException
     * @throws IOException
     */
    @Override
    public void service(ServletRequest servletRequest, ServletResponse servletResponse) throws ServletException, IOException {
        System.out.println("service方法执行");
    }

    @Override
    public String getServletInfo() {
        return null;
    }

    /**
     * 当servlet对象被销毁时执行该方法
     * 服务器停止运行时, 对象被销毁
     * 做一些资源释放的操作
     */
    @Override
    public void destroy() {
        System.out.println("helloServlet被销毁");
    }
  }

# 2. 通过继承实现 －－＞

继承来实现，设计模式上属于适配器模式
适用场景：接口过多，但是常用的方法很少
优点：不常用的方法

继承父类:

# 2.1 继承 GenericServlet

常用于重写 service 方法

实例

public class MainServlet  extends GenericServlet {

    @Override
    public void service (ServletRequest servletRequest, ServletResponse servletResponse) throws ServletException, IOException {
        // 给页面显示一点内容
        System.out.println("service方法");
        // 开启一个字符流
        PrintWriter writer = servletResponse.getWriter();
        writer.println("<h1>hello servlet</h1>");
        // 关闭字符流
        writer.close();

或者

# 2.2 继承 HttpServlet

重写 doPost doGet 方法

继承HttpServlet更常用,使用时注意删除super.doGet ; super.doGet

实例


// @WebServlet -- 快速配置 不用在xml中再配置了
// !!!!url 中的"/"要记得写,不然很多的请求中报错很难找
// !!!url中的大小写有要求,会按照大小写区分
@WebServlet(urlPatterns = "/home")
public class IntroServlet  extends HttpServlet {

    @Override
    protected void doGet(HttpServletRequest req,
                         HttpServletResponse resp) throws ServletException, IOException {
         // super.doGet(req, resp);  必须删掉 不然会不支持这个请求
                 
         // 在头部文件指明发送给浏览器的内容以及格式
         // resp设置响应头, 方便浏览器直接判断响应的内容
        resp.setHeader("Content-Type", "text/html;charset=utf-8");

        ServletContext context = this.getServletContext();

        // 读取context中的数据
        Object name = context.getAttribute("name");
        // 需要先访问存值的页面, 再访问取值的页面 才能取到context中的值
        System.out.println("从context中读取的数据"+name);
        // 删除context保存的数据
        context.removeAttribute("name");
        System.out.println("删除context的数据");
        PrintWriter pw = resp.getWriter();
        pw.write("<div>第一个请求</div>");
        pw.close();
    }

    @Override
    protected void doPost(HttpServletRequest req,
                          HttpServletResponse resp) throws ServletException, IOException {
          // 从父类继承过来的doPost方法需要删除
          // ?? : 父类中默认不支持 doPost 和 dpGet
//        super.doPost(req, resp);
    }
}

# Servlet 的常用三大对象 (requset ; session ; servletContext)

获取方法

三大常用对象，
都是通过 request 调用 get 方法来得到一个对象

区别

种类 类名 生命周期
request HttpServletRequest 本次请求有效
session HttpSession 当前浏览器有效，可以跨页面
servletContext ServletContext 整个项目有效，可以跨页面

``

# servletContext

一个项目只有一个 servletContext 对象 (全局)
在项目启动期间，保存在 servletContext 中的数据可以一直使用 (跨页面)

生命周期
项目启动到关闭，服务器重启时 会被销毁
