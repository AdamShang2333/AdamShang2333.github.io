# 外键 (foreign key)

外键：描述与其他表的关系

  1. 一对一
  2. 一对多 --> 先创建 "一"
    一个部门对应多个员工，那么就在员工表中设置一个外键 引用部门表的主键
  3. 多对多 --> 创建一个中间表
# 外键
# 1. 外键只和 DML(增删改) 有关系
# 2. 只有当被关联表(部门表)中有相应的值,才能在员工表中insert,update
# 3. update 和 delete 都会引起相应的数据库问题 (更新风暴和删除时将不想删的数据一起删掉)
#  所以正常来说 on update / delete cascade都不常用
# !!!!改的操作容易引起更新风暴()
# 4. 如果删除表 , 一定要先删除emp(员工表 "多") 再删除dept(部门表"一")
		--> !!!!确保没有外键的数据需要依赖主键,主键才能删除数据
# 5. 如果在创建表时, emp表中添加了on update cascade,
#     那么emp表中的外键可以随着dept表中的主键改变
#   如果在创建表时不添加,那么dept表中的主键不能随意修改
# 6. 如果emp表的外键添加了 on delete cascade
#     那么删除dept表中数据时 会把emp表中所有外键对应的行全部删除
#     如果不添加 那么只有在dept的主键没有任何的emp外键对应的情况下
#     dept中的数据才能被删除

# 一对多

主键表 (一) 的删除会同步到外键表 (多)

制表计划

# 员工表
# name age gender workage salary 部门id(外键)
# 部门表
# 部门id(主键) 名字 领导 创立时间 部门职责 部门描述...

功能实现

#部门表
create table if not exists dept
(
  id   integer primary key auto_increment comment '部门id',
  name varchar(10) comment '部门名称',
  des  varchar(10) comment '部门描述'
);
# 员工表添加外键  外键为蓝色钥匙
create table if not exists emp
(
  eid     integer primary key auto_increment comment '员工id',
  name    varchar(10) comment '员工姓名',
  dept_id integer comment '部门id',
  age     integer,
  salary  integer,
  foreign key (dept_id) references dept (id)
    #部门表的删改会同步到员工表中
    on update cascade
    #可以随便删除 部门删掉 员工表同步删除
    on delete cascade
) comment '员工表' charset = 'utf8';

# 多对多

# 笛卡尔积

将表 1 的 n 条数据和表 2 的 m 条数据所有情况组合在一起 (n*m 条数据)
多表查询本质上就是对两表结果 (笛卡尔积) 的过滤

select emp.name ename, dept.name dname
from emp,
     dept
where emp.dept_id = dept.id;

# 笛卡尔积过滤

过滤使用 on 子句
ON where having三个条件过滤子句的执行顺序(不添加()提升优先级的情况下):
ON > where > having

# 内连接 (inner join 使用时可省略 inner 默认为内连接)

select *
from emp
       inner join dept
  # on 专门过滤笛卡尔积
       on emp.dept_id = dept.id;

# 左连接 (LEFT JOIN)

--> join 左面的表无论是否满足 on 的条件，所有的行都会留在结果中
　　(右表不满足的在右边留一行空的列，即左与 null"拼接")

# 右连接 (RIGHT JOIN)

-->-join 右面的表无论是否满足 on 的条件，所有的行都会留在结果中
　　(左表不满足的在左边留一行空的列，即右表与 null"拼接")
